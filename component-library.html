Envision Design System · Components

Component Library

The individual building blocks. Buttons, cards, stats, form fields, callouts, chips, dividers, eyebrows. Use these to assemble the section patterns.

Component 01

Buttons

Uppercase, tracked at 0.25em, Bold. Three variants: Primary (Green) for conversion CTAs, Secondary (Blue) for navigation-level actions, Tertiary (ghost) for secondary options. On dark backgrounds, use the ghost-on-dark variant.

Secondary — Blue — Navigation-level actions
Tertiary — Ghost — Secondary options
On dark backgrounds
Component 02

Eyebrows

The small uppercase label that introduces a headline. Three variants. Standard (green, 0.25em) is the default. Pillar (green, 0.5em — wider, more editorial) for program pillar sections. Neutral (charcoal) for utility contexts.

Component 03

Cards

Used in feature grids, program summaries, capability breakdowns. Visual identifier is a 3px colored top border — no drop shadows, no hover lifts. Optional accent variants (green, goldenrod) draw the eye to one featured card per grid.

Component 04

Stat displays

Large number, small label. Numbers use display type weight and tight tracking. On light backgrounds, Primary Blue by default; one stat may use Primary Green as the hero metric. On dark backgrounds, white numbers with the same single-green-accent rule.

Component 05

Callout boxes

Inline informational callouts. Left border 4px in a contextually-appropriate color. Background is a very-subtle tint of the border color. Used for sidenotes, process callouts, and highlights inside long-form content.

Component 06

Chips / tags

Small labels for metadata, category identifiers, certification markers. Uppercase, tracked at 0.15em. Four variants: Primary (filled blue), Green (filled green), Outline (bordered blue), Subtle (neutral bordered).

Component 07

Form fields

Label in uppercase eyebrow style above the input. Inputs are bordered with subtle gray, 4px radius, 14-16px padding. Focus state: border goes to Primary Blue. Form fields live in a 2-column grid on desktop, stack on mobile.

Component 08

Dividers

Thin horizontal rules that separate content within a section. Three variants. Subtle (1px gray) for most uses. Strong (2px blue) to mark major content boundaries. Accent (3px short green) as a decorative mark beneath a headline.

Subtle — default separation inside content
Strong — major section boundary
Accent — mark beneath a headline
Component 09

Type samples in context

How typography looks inside real content blocks. Always title case for headlines. Always tracked per the brand book. Body copy at -0.02em for subtle optical refinement.
