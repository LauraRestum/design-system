Section Patterns — Ten Building Blocks That Make an Envision Page

Most Envision pages are assembled from these ten section types. Study them, then compose. Each pattern is preceded by a note explaining when to use it, what makes it on-brand, and what breaks it.

  1. Hero, type-forward
  2. Hero, split with image
  3. Stat block (3-4 column)
  4. Feature grid — cards
  5. Pull quote / testimonial
  6. Pillar / program section
  7. Full-width callout
  8. Data-forward section
  9. Image grid / gallery
  10. Footer
Pattern 01

Hero, type-forward

Use for: Commercial sales pages, B2B landing pages, federal work Background: Solid Primary Blue Height: 128-160px vertical padding

The most Envision-looking hero. Type does all the work. No image, no illustration, no gradient — just big confident typography on solid navy or primary blue, a supporting subline, and a single CTA. Green shows up twice: the eyebrow and the button.

✓ Use when: You need enterprise credibility. When photography isn't available. When the headline is the message and nothing else matters.

✗ Never: add drop shadows, gradient overlays, decorative lines, or illustrations. Let the type stand alone.

Pattern 02

Hero, split (type + image)

Use for: Program pages, mission-forward landing pages Layout: 7/5 grid, content left, image right Height: 128px vertical padding

The workhorse of mission and program pages. Content and image share the hero. Photography goes on the right at a 4:5 vertical aspect ratio, showing a person in action. The image never gets a rounded mask or decorative overlay. Sharp corners, sharp edges.

✓ Use when: A person or moment is central to the story. Program pages, pillar pages, donor-facing work.

✗ Never: use generic stock photography. Never apply a circular crop. Never let the image outweigh the headline visually.

Pattern 03

Stat block (3-4 column)

Use for: Capability proof points, impact summaries, by-the-numbers blocks Layout: 3 or 4 equal columns Background: Off-white or white

Large numbers dominate. Labels are small and uppercase. One stat may use Primary Green to highlight the hero metric. Never more than one green stat — it loses emphasis.

✓ Use when: You have real, verified numbers. Anywhere proof points are stronger than copy.

✗ Never: make all the numbers green. Never use fake or vague stats ("100s of…", "many…").

Pattern 04

Feature grid — cards

Use for: Service lists, capability breakdowns, program summaries Layout: 3 columns on desktop, 1 on mobile Background: White; cards on off-white

Cards have a 3px colored top border as the only visual identifier — no drop shadows, no hover lifts, no decorative icons unless content requires them. One card may use the green top border to draw the eye.

✓ Use when: You have 3-6 parallel items of equal importance.

✗ Never: use more than 6 cards — break into two sections instead. Never use shadows to create depth.

Pattern 05

Pull quote / testimonial

Use for: Customer quotes, client testimonials, mission-driven moments Alignment: Centered Height: 128px vertical padding minimum

Quote set in Primary Blue at 28-36px, Bold, tight line-height. Attribution in uppercase caption style beneath. Plenty of whitespace — this section only does one thing, let it breathe.

✓ Use when: A direct quote moves the story forward better than description could. For mission-driven pages.

✗ Never: decorate with quotation marks, scrolls, or illustrations. The white space is the decoration.

Pattern 06

Pillar / program section

Use for: Introducing one of the eight program pillars Layout: 1/1 split Eyebrow tracking: 0.5em uppercase

This uses the large-subhead treatment — uppercase at 0.5em tracking — as the eyebrow label, giving each pillar an editorial, magazine-cover feel. The visual can be photography or an accent-color block. Extended palette colors are welcome here to distinguish pillars.

✓ Use when: Introducing a specific pillar (Arts, Research, Workforce, Mission Services, etc.).

✗ Never: use the same pillar eyebrow tracking on non-pillar sections. It dilutes the pattern.

Pattern 07

Full-width callout

Use for: Pre-footer CTAs, major section breaks Background: Primary Blue, Navy, or Primary Green Height: 96px vertical padding

A full-bleed band of solid color that interrupts the normal page flow. One headline, one CTA. When the background is Primary Green, the headline goes Primary Blue for contrast (one of the eight approved WCAG combinations).

✓ Use when: You want a decisive break before the footer, or to separate major sections with a confident moment.

✗ Never: use more than one full-width callout per page — they lose impact if repeated.

Pattern 08

Data-forward section

Use for: Capability tables, certification displays, contract vehicles Background: White Primary use: Commercial/federal work

For procurement-facing pages. Dense, credible, type-dominant. Tables use Primary Blue rules (top border 2px, row borders 1px subtle gray). Certifications appear as bordered tags in Primary Blue. One data cell may highlight in Primary Green.

✓ Use when: Your audience needs to verify specifics — spec tables, contract vehicles, certifications.

✗ Never: use emoji, colored row backgrounds, or decorative icons in data sections. Let the data be credible.

Pattern 09

Image grid / gallery

Use for: Event recaps, program showcases, facility tours Layout: 2-3 columns, sharp rectangles Gap: 24px

Photography shown at consistent aspect ratios, sharp rectangles, 4px max corner radius. Caption or label optional, placed as a lower overlay. Use for moments where the visual variety IS the message — a facility, an event, a program in action.

✓ Use when: Multiple photos collectively tell a story that one image couldn't.

✗ Never: use circular photo crops, round masks, or Instagram-style filters. Straight photography, treated with restraint.

Pattern 10

Footer

Use for: Every page Background: Navy (#002855) Layout: 4 columns on desktop

Navy background (darker than hero blue, so it reads as "bottom" of the page). Primary Green uppercase eyebrow labels for each column. Body links in secondary-on-dark color. A thin divider separates the main columns from the copyright line.

✓ Use: 4-column structure at desktop — brand area (2 cols), then 3 single-column nav lists.

✗ Never: use Primary Blue for the footer (too similar to the hero — no visual differentiation). Never use social-media icon clutter.
